Terminal Commands

cd


This command changes the directory that you are currently working with in the terminal. This is a very useful and common command, which let you create delete and edit files in different folders.

ls


This command is used to check which files and directories exist in a certain folder/area. You can use this command after cd to check the contents of your current location.

code .


This command is useful when working with vscode, as this will open the current directory in visual studio code.

cp


This command is used to copy a file from one location to another. This can also be used to just copy a file in the same directory with a new name. When specifying the first argument, include the originating file that you wish to copy followed by a space and a full path, filename and extension of where you want the copy to be placed when the command is executed. The Terminal will return when the copy has been completed.

mv


When you want to not copy a file but move it, use this command instead. It has the same format as cp.

mkdir


This command lets you create a directory right from your CLI. This is very convenient in many situations. You can specify a path if you want the folder to be created somewhere else than your current location.

rmdir


This command simply lets you delete a directory.

sudo


This command allows you to elevate your user privileges while executing the command to administrator privileges. This can be required in some situations.

clear


This simply clears the terminal.

npm install


Installs packages in the package.json file in the local node_modules folder.

npm uninstall


Remove a package from the package.json file and removes the module from the local node_modules folder. You have to specify which package, or it will remove all packages.

npm update


This command updates the specified package. If no package is specified then it updates all the packages in the specified location.

npm update -g


This command will apply the update action to each globally installed package.

npm init


Creates a package.json file in our directory. It basically asks some questions and finally creates a package.json file in the current project directory.

npm start


Runs a command that is defined in the start property in the scripts. If not defined it will run the node server.js command.

npm build


It is used to build a package.

npm ls


Lists all the packages as well as their dependencies installed.

npm version


Bumps a package version.

npm help


Searches npm help documentation for a specified topic. It is used whenever the user needs help to get some reference.
